RULE 1: FASTAPI FORM FIELDS
═══════════════════════════════════════

═══════════════════════════════════════
RULE 3: PILLOW TEXT MEASUREMENT
═══════════════════════════════════════
draw.textsize() was REMOVED in Pillow 10+. NEVER use it.

WRONG:
    w, h = draw.textsize(text, font=font)

CORRECT:
    bbox = font.getbbox(text)
    w = bbox[2] - bbox[0]
    h = bbox[3] - bbox[1]

═══════════════════════════════════════
RULE 4: CACHE KEYS
═══════════════════════════════════════
Python's hash() is non-deterministic across processes. NEVER use it for cache keys.

WRONG:
    cache_key = f"caption:{hash(top + bottom)}"

CORRECT:
    import hashlib
    cache_key = f"caption:{hashlib.sha256((top + bottom).encode()).hexdigest()}"

═══════════════════════════════════════
RULE 5: CORS CONFIGURATION
═══════════════════════════════════════
allow_origins=["*"] combined with allow_credentials=True is REJECTED by all browsers.

WRONG:
    allow_origins=["*"]
    allow_credentials=True

CORRECT — pick one:
    Option A (development): allow_origins=["*"], allow_credentials=False
    Option B (production):  allow_origins=["https://yourdomain.com"], allow_credentials=True

═══════════════════════════════════════
RULE 6: FILE SIZE VALIDATION
═══════════════════════════════════════
Never just comment out file size checks. ALWAYS implement them in code.

CORRECT:

    contents = await file.read(MAX_SIZE + 1)

        raise HTTPException(400, "File too large — max 10MB")

═══════════════════════════════════════
RULE 7: ASYNC AND BLOCKING I/O
═══════════════════════════════════════
Pillow and file operations BLOCK the async event loop. Always offload them.

WRONG:
    result = add_text_to_image(img_bytes, top, bottom)

CORRECT:
    import asyncio
    loop = asyncio.get_event_loop()
    result = await loop.run_in_executor(None, add_text_to_image, img_bytes, top, bottom)

═══════════════════════════════════════
RULE: NEVER BLOCK THE ASYNC EVENT LOOP
═══════════════════════════════════════
Any CPU-heavy or I/O operation inside an async def MUST use run_in_executor.
This includes: Pillow, file reading, PDF processing, image resizing.

WRONG:
    async def endpoint():
        img = Image.open(file)  # blocks event loop

CORRECT:
    import asyncio
    from functools import partial

    def process(image_bytes):
        img = Image.open(io.BytesIO(image_bytes))
        # all pillow work here
        return img

    async def endpoint():
        image_bytes = await image.read()
        loop = asyncio.get_event_loop()
        img = await loop.run_in_executor(None, process, image_bytes)

═══════════════════════════════════════
RULE: ALWAYS VALIDATE FILE UPLOADS
═══════════════════════════════════════
Every upload endpoint MUST check file size AND file type before processing.

CORRECT — add this at the START of every upload handler:
    MAX_SIZE = 10 * 1024 * 1024  # 10MB
    ALLOWED_TYPES = {"image/jpeg", "image/png", "image/webp"}

    image_bytes = await image.read(MAX_SIZE + 1)
    if len(image_bytes) > MAX_SIZE:
        raise HTTPException(400, "File too large — max 10MB")
    if image.content_type not in ALLOWED_TYPES:
        raise HTTPException(400, f"Unsupported type: {image.content_type}")
    # only THEN process the image

RULE: ALWAYS HANDLE FETCH ERRORS
═══════════════════════════════════════
Every fetch() call MUST have try/catch and check res.ok.

WRONG:
    const res = await fetch(url, options)
    const blob = await res.blob()

CORRECT:
    try {
        const res = await fetch(url, options)
        if (!res.ok) throw new Error(`Server error: ${res.status}`)
        const blob = await res.blob()
    } catch (err) {
        setError(err.message)
    } finally {
        setLoading(false)
    }
